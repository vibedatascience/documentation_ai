Lecture 1: Introduction to neural networks and deep learning

The problem

  • AI was founded at the 1956 Dartmouth workshop. Minsky, McCarthy, and Shannon were all MIT.
  • The decades since have produced a sequence of breakthroughs, each one an answer to the previous approach's failure:
traditional AI interview experts, hand-write if-then rules │ fails: Polanyi's paradox, no generalization ▼ machine learning learn input→output mapping from examples │ fails on unstructured data (images, text, audio) ▼ deep learning learns the representation itself, automatically │ ▼ generative AI deep learning that produces unstructured output
  • Traditional AI: ask human experts how they do the task, write if-then rules, program the rules. Fails twice. Polanyi's paradox: we know more than we can tell. And hand-written rules cannot cover situations nobody anticipated.
  • Machine learning: give the computer input-output examples and learn the mapping statistically. Huge impact (credit scoring, loan granting, disease prediction, demand forecasting), but only where the input is structured.
  • Structured data: data you can numericalize into a spreadsheet.
  • The raw form of unstructured data has no intrinsic meaning. An image is three tables of 0-255 numbers for red, green, and blue light. A 251 in the blue table says nothing about what the object is.
  • The workaround was to manually build a representation first (feature extraction). Massive human effort, and this human bottleneck sharply limited the reach of ML.

The idea

  • Deep learning extracts representations automatically from raw unstructured data, then feeds them to what is basically linear or logistic regression at the end.
  • This demolishes the human bottleneck.
  • The breakthrough came from three forces: new algorithmic ideas, unprecedented data from the digitization of everything, and GPU compute, applied to an old ML idea, neural networks.
  • Immediate application: every sensor in the world can be given the ability to detect, recognize, and classify what it is sensing.

What is a neural network?

  • Start at the beginning: recall logistic regression. Inputs [x1, x2, ..., xk] go in, a probability comes out.
  • Worked example: predict who gets called for a job interview from GPA and experience. Fitting the model gives intercept 0.4 and coefficients 0.2 (GPA) and 0.5 (experience).
  • The formula can be rewritten as a network of mathematical operations: input data flowing through two functions that have been connected. First a linear sum, then the sigmoid. Data flows left to right.

P(interview) = σ(0.4 + 0.2·GPA + 0.5·exp) = 1 1 + e−(0.4 + 0.2·GPA + 0.5·exp)

GPA ────0.2───┐ ├──▶ (+) ──▶ σ ──▶ P(interview) exp ────0.5───┘ ▲ │ 0.4 (bias)
  • Terminology swap: coefficients are weights, the intercept is a bias.
KEY IDEA · WHY THE NETWORK LENS
The moment you write the model as a network, you realize you can do lots of things in the middle. You do not have to go from input straight to output. The middle is where the automatic representation learning will come from.

Things in the middle

  • The simplest thing to insert: a stack of linear functions (a circle with a plus is shorthand for one). k inputs, k could be 1,000, get compressed into, say, 3 numbers. Maybe 4 is right, maybe 10. How to choose comes later.
  • To learn smart representations, transform the inputs one or more times before doing the prediction. Flow the 3-dimensional vector through another function f(x):
The network, term by term (click a term, or let it cycle)
  • f(x) is called an activation function and it must be nonlinear, otherwise all the effort is a total waste of time. Take it on faith for now, the why comes in a few minutes.
  • Repeat as many times as you want, 100 if you like. Every transformation gives the network capacity to learn something interesting from the data. Capacity, not a guarantee.
DEFINITION · NEURAL NETWORK
A neural network is nothing more than repeatedly transformed inputs which are finally fed to a linear or logistic regression model. The input stays the same, the output stays the same, only the middle changes.
  • Explainability: for any input you know exactly how it flows through the network. What you cannot do is assign credit. The attributes get enmeshed in the middle, so no single input can be blamed for the outcome.
  • The number of layers and the functions inside them are design choices. The problem fixes the input and the output, we choose the middle.
  • Terminology: linear followed by nonlinear is a neuron (the brain link is loose and heavily argued, ignore it in practice). A vertical stack of neurons is a layer. The input layer is just the input, the output layer produces the output, everything between is a hidden layer.

Activation functions

  • An activation function receives a single number and outputs a single number. The linear function before it collapses 10 or 1,000 inputs into one scalar, and the activation transforms that scalar. That is all it is.
  • Sigmoid: 1 / (1 + e−a). Very negative inputs come out near 0, very large inputs come out near 1, and all the action happens in the middle range.
  • Linear activation: passes the number straight through. Barely deserves the name, listed for completeness.
  • ReLU (rectified linear unit), the hero of deep learning: max(x, 0). Positive numbers pass unchanged, negative numbers get squashed to 0. Many longtime practitioners believe ReLU is one of the key factors behind deep learning's success.
  • There is a menagerie of others (tanh, leaky ReLU, GELU, swish), but ReLU in the hidden layers gets practically anything done.
  • Rule of thumb: ReLU for hidden layers. The output layer is not a choice, the nature of the output dictates it. A single probability needs a sigmoid, 10 probabilities that must sum to 1 need a softmax.

Counting parameters

  • The first real network for the interview problem: 2 inputs (GPA, experience), one hidden layer of 3 ReLU neurons, a sigmoid output. Concept check from class: how many parameters does it have? Guesses were 12 and 9. The answer is 13, and the reason everyone says 12 is that they forget the biases.
  • The recipe. Weights: every arrow between consecutive layers is one weight, so multiply the layer sizes. Biases: every neuron that computes something has one, so count the neurons (the input layer computes nothing).
hidden: 2 inputs x 3 neurons = 6 weights + 3 biases = 9 output: 3 inputs x 1 neuron = 3 weights + 1 bias = 4 total 13
  • General rule for any layer: nin × nout weights plus nout biases. Sum over layers.
  • Habit to build: hand-calculate the parameter count the first two or three times you meet a complex network. Once you get it right a couple of times you never need to again, but it proves you understand what is connected to what.
