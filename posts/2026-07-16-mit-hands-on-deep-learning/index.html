MIT 15.773 Hands-On Deep Learning: notes on all 11 lectures

MIT put the full lecture videos of 15.773 Hands-On Deep Learning (Sloan, Spring 2024, Prof. Rama Ramakrishnan) on OpenCourseWare. Eleven lectures, about 76 minutes each. The course goes from a single neuron to fine-tuning LLMs with LoRA and generating images with Stable Diffusion, and every concept lands in a Keras Colab before the lecture ends. I pulled the subtitles for all eleven videos and wrote up what each one actually covers. These notes are my compression of roughly 140k words of transcript, in my words. Watch the videos for the Colab walkthroughs.

Prereqs are light: Python plus ML 101 concepts (train/test splits, overfitting, regularization). The teaching style is socratic. Ramakrishnan asks the class to hand-count parameters, guess design choices, and spot flaws before he reveals them, and the transcripts are full of student questions that anticipate the next topic.

Lecture 1: What deep learning is, and a neural net as a graph

The framing: AI has had three breakthroughs since 1956. Rule-based systems failed because of Polanyi's paradox (we know more than we can tell, so experts cannot write down the rules they actually use) and because hand-written rules do not generalize to edge cases. Machine learning replaced rules with input-output examples, but it only works on structured data, meaning data you can put in spreadsheet columns. Images and text are unstructured: a pixel value of 251 says nothing about whether it depicts sky, water, or paint. Historically people bridged this with manual feature engineering (measure the beak length, the wingspan), and entire PhDs went into hand-crafting representations for one narrow task. Deep learning's contribution is that it learns the representations itself, and that is the single idea the whole course hangs on.

The mechanics start from logistic regression, redrawn as a network: inputs flow left to right, get multiplied by coefficients, summed, and squashed through a sigmoid. Coefficients are called weights, intercepts are called biases (so "leaked model weights" in the news means the coefficients got out). The point of the graphical form is that it makes an obvious move visible: you can insert layers of linear functions between input and output. Stacked linear functions collapse to one linear function, so each hidden node gets a nonlinearity, and the practical default is the ReLU, max(0, x). The output layer is dictated by the problem (sigmoid for a probability, softmax for multi-class); everything in the middle is a design choice.

The lecture closes with the discipline he repeats all term: hand-count the parameters. A 2-input, 3-ReLU-hidden, 1-sigmoid network has 13 parameters (2x3 + 3 weights, plus 4 biases everyone forgets). Written out as a single formula, even this toy network is unreadable, and that complexity is where the capability comes from.

Lecture 2: Training. Loss functions and gradient descent

Case study: Cleveland Clinic heart disease data. 303 patients, 13 variables (29 after one-hot encoding the categoricals), binary label for a diagnosis within a year. Design choices: one hidden layer of 16 ReLUs (he tried 4, 8, 16; above 16 it overfit), sigmoid output, 497 parameters. The number of weights between two layers is roughly the product of their sizes, which is why parameter counts explode and why overfitting is a standing concern.

Training is defined as minimizing a loss function, a measure of how bad the model is. Regression gets mean squared error; binary classification gets binary cross-entropy, which punishes confident wrong predictions with a loss that grows like negative log of the predicted probability. These specific functions are chosen because they are mathematically well behaved and have good gradients, not because they are the only options (he points to pinball loss for asymmetric penalties).

Then the optimization core. Gradient descent is just: compute the derivative, step opposite to it, scaled by a learning rate. w_new = w_old minus alpha times gradient. Invented by Cauchy in 1847, and it is essentially the algorithm that trained GPT-4, which he finds astonishing and says so. The learning rate is small (0.1, 0.01) because the gradient is only valid locally, and published papers treat their learning-rate schedules as part of the IP. Stochastic gradient descent computes the gradient on a random minibatch (say 32 rows) instead of the whole dataset: one update per batch, not per epoch. It is only an approximation of the true gradient, and that noise is a feature, because it lets the optimizer escape local minima of the full loss. Backpropagation is the chain rule organized layer by layer from the output backwards, so intermediate quantities get reused instead of recomputed. Adam, an SGD variant, is the course default optimizer.

Lecture 3: Keras end to end, then multi-class with softmax

First half formalizes the vocabulary you need to read any training log. An epoch is one full pass through the training data. With SGD the epoch is split into batches (194 training rows at batch size 32 gives 7 batches, the last one small, which is fine), and weights update after every batch. Batch sizes are powers of 2 because of GPU alignment.

Then the heart disease Colab, which is 80% preprocessing and deliberately so: check class balance first (72.6% negative, so the do-nothing baseline is 72.6% accuracy and any model must beat it), one-hot encode categoricals with pd.get_dummies because neural nets take only numbers, split train/test BEFORE standardizing so test-set statistics never touch the pipeline, standardize numerics to mean 0 and unit variance because networks train best on small input ranges, convert to NumPy, and only then model.compile(optimizer='adam', loss='binary_crossentropy', metrics=['accuracy']) and model.fit. The compile step reorganizes the graph for parallel hardware. He verifies model.summary()'s 497 parameters by hand on the board.

Second half moves to Fashion-MNIST: 60,000 grayscale 28x28 clothing images in 10 classes. Flatten each image to a 784-long vector, feed a 256-ReLU hidden layer, and the new problem is the output: ten mutually exclusive classes need ten probabilities that sum to 1, which ten independent sigmoids cannot give you. The softmax does it by exponentiating each logit and dividing by the total. He connects it forward: GPT-4 emits every token via a roughly 50,000-way softmax over its vocabulary. Loss function bookkeeping: binary_crossentropy for 0/1 labels, sparse_categorical_crossentropy for integer class labels, categorical_crossentropy for one-hot labels. Same math, different data encodings.

Lecture 4: Convolutions, pooling, and transfer learning

Why flattening images into dense layers is wrong, three ways: parameter explosion (a 27-million-number iPhone photo into a 100-neuron dense layer is 2.7 billion weights), loss of spatial adjacency (neighboring pixels are related and the flat vector forgets it), and no translation invariance (a vertical line detector should work anywhere in the image, not be relearned per location).

The convolutional filter fixes all three. It is a small matrix, say 3x3, slid across the image; at each position you multiply matching numbers, sum, and pass through a ReLU. With the right numbers a filter detects a horizontal line, a vertical line, a color gradient. The punchline to "where do the numbers come from" is the same as always: random init plus backprop. A convolutional layer is a set of such filters, so the layer's depth is the filter count, a design choice. Pooling (max pooling) shrinks height and width by taking the max of each 2x2 patch, and the intuition offered is an OR gate: did anything light up in this region? Stacked conv blocks (1-2 conv layers plus pooling) get spatially smaller and deeper as you go; ResNet turns a 224x224x3 image into 7x7x2048. Early layers learn lines and edges, later layers learn honeycombs, torsos, and whole Labrador faces, shown via the classic feature visualization paper. The depth grows because atomic features are few but their combinations are combinatorial, like molecules from atoms.

The applied half is the part to steal: transfer learning. Take ResNet trained on ImageNet, chop off the final 1000-way layer, run your images through the headless network, and treat the output tensor as a smart representation. A tiny dense head on top hits 100% test accuracy classifying handbags versus shoes from 98 training examples, demonstrated live with a webcam. Data augmentation (random flips, rotations, zooms as Keras layers) stretches small datasets, with the caveat that augmentation must not change meaning: horizontal flips yes, upside-down dogs no. Pretrained models live on TensorFlow Hub, PyTorch Hub, and Hugging Face, which was at half a million models when he checked.

Lecture 5: NLP begins. Vectorization and bag of words

The motivation section argues that text-in text-out, as a formalism, covers classification, entity extraction, summarization, generation, code, and question answering, and that call centers alone are a giant application. Then the mechanics. Text must become numbers, via a pipeline he shortens to STIE: standardize (modern practice is just lowercase and strip punctuation; nobody bothers stemming or removing stop words anymore), tokenize on whitespace, index words to integers against a vocabulary, encode as one-hot vectors. Cap the vocabulary at the most frequent words (roughly 50k covers almost everything in English); anything unseen at inference maps to a shared UNK token, which is lossy since Hamlet and Romeo become indistinguishable.

A sentence is then a table of one-hot rows, and the bag of words model collapses the table to one fixed-length vector, either multi-hot (is the word present) or count encoding. Word order is destroyed ("the cat sat on the mat" equals "the mat sat on the cat"), yet for many tasks it does not matter. The Colab classifies song lyrics into hip hop, rock, or pop: multi-hot input, one ReLU hidden layer, 3-way softmax, about 72% accuracy. Moving to bigrams (adjacent word pairs, ngrams=2 in Keras, vocabulary raised to 20k) restores a bit of local order and lifts it to 75%.

Dropout gets its proper introduction here: a layer that randomly zeroes activations during training so no neuron can rely on another, breaking the collusion that produces overfitting. The origin story he tells, possibly apocryphal, is Hinton noticing banks rotate branch staff to prevent collusion fraud. Hyperparameter search (neurons, learning rate, activations) is trial and error, or KerasTuner if you want it systematic.

Lecture 6: Embeddings, or giving words geometry

One-hot vectors have two fatal problems: they are as long as the vocabulary, and every pair of words sits at distance root 2 from every other, so good/bad are exactly as far apart as movie/film. There is no notion of meaning in the geometry at all. The fix is dense, low-dimensional, learned vectors where geometry encodes semantics: buildings cluster with buildings, fruit with fruit, and directions carry meaning too, so the vector from puppy to dog is the same "grows up" direction as calf to cow, and man minus brother plus woman lands near sister. That is word vector algebra, and it falls out of training without anyone asking for it.

How they are learned, GloVe style: build a co-occurrence matrix over a giant corpus (how often does each word pair appear near each other), then find vectors and biases whose dot products reproduce the log co-occurrence counts, minimizing squared error by gradient descent. Nothing about it is a neural network; SGD works on any differentiable function. The embedding length (50, 100, 300) is a capacity dial, and the dimensions are not interpretable: nobody, including the authors, knows what dimension 37 means, only that the whole vector reconstructs co-occurrence well. Polysemy is the known weakness: "bank" gets one averaged vector across all its senses, which is the setup for contextual embeddings next week.

The Colab runs the genre classifier three ways and produces the lecture's practical rule. Frozen pretrained GloVe: 63%. GloVe fine-tuned with trainable=True: 69%. Embeddings trained from scratch for the task: 72%. Pretrained embeddings shine when your data is small (like ResNet did for 98 handbags); with 50,000 labeled examples, training your own wins.

Lecture 7: Self-attention and the transformer encoder

Motivating task: slot filling on the ATIS airline query dataset. Every word of "i want to fly from boston at 7 am" gets one of 123 labels (departure city, arrival time, other), with B/I prefixes marking multi-word entities. So the architecture must emit one output per input token, handle variable-length inputs, and use each word's surrounding context. Transformers deliver all three, which is why they now run translation, search, speech, vision, protein folding, and every LLM.

Self-attention in its stripped-down form: to contextualize a word, take the dot product of its embedding with every word's embedding in the sentence (including itself), softmax those scores into weights that sum to 1, and output the weighted average of all the embeddings. "Station" in a sentence containing "train" gets pulled toward the train sense; "radio" is absent so contributes nothing. Standalone embeddings in, contextual embeddings out, same count and dimension. Word order enters separately: each position (1st, 2nd, 3rd...) gets its own learned embedding, added element-wise to the word embedding before the stack. Multi-head attention runs several attention computations in parallel because a sentence has many simultaneous patterns worth attending to (meaning, grammar, tense), exactly like multiple filters in a conv layer; the heads' outputs are concatenated and projected back to the original width by a dense layer.

The Colab builds the full slot filler in Keras: 30-token max sequences, 512-dim embeddings, 5 heads, one transformer block, then a ReLU and a 125-way softmax per token. A sharp student notices the trap he planted: plain dot-product attention has no learnable parameters at all, so what is training even doing? Answer deferred to lecture 8. The model hits 99% token accuracy, 93% on non-trivial slots.

Lecture 8: The real transformer. Q, K, V and friends

Billed as the most demanding 30 minutes of the course. Three upgrades turn toy self-attention into the 2017 paper. First, everything vectorizes: stack the embeddings into a matrix X, and all pairwise dot products are just X times X-transpose, softmaxed row-wise, times X again. That single matrix formulation is what makes transformers GPU-friendly, and without it there is no transformer revolution. Second, the answer to lecture 7's cliffhanger: before the dot products, the input is linearly transformed by three learned matrices into queries (the word doing the looking), keys (the words being compared against), and values (what gets averaged). Those A_Q, A_K, A_V matrices are the learnable parameters of an attention head, and each head has its own randomly initialized set, which is why heads diverge and learn different patterns. Third, the plumbing that makes deep stacks trainable: residual connections (add the block's input back to its output, so gradients and information flow past every layer) and layer normalization (rescale activations to a well-behaved range). Blocks preserve shape, so you stack them; GPT-3 stacks 96.

A consequential aside: parameter count is independent of sentence length. The A matrices only care about embedding width, and length shows up solely in the positional table, which is how million-token context windows exist without more weights.

Applications section: for whole-sequence classification, prepend an artificial CLS token and classify its contextual embedding rather than crudely averaging everything. BERT is this encoder trained by masking, covered via homework and Hugging Face pipelines (sentiment, QA with answer spans you can audit against the source text). Then the portability argument: the block is never modified for new domains, only the tokenization is. Vision Transformer chops an image into patches, flattens each, projects through a dense layer, adds positional embeddings, done. TabTransformer embeds categorical columns and concatenates continuous ones at the end. Multimodal models embed text, images, and tables into one shared sequence and let attention sort out that this word relates to that image patch.

Lecture 9: From BERT to GPT. Causal language models

Next-word prediction is masking where the blank is always at the end. Training data is free: shift the sentence one position left, so "the cat sat on the" predicts "cat sat on the mat" position by position. The catch a student spots immediately is that vanilla self-attention peeks at the future; the network would trivially copy the answer from the input. The fix is causal (masked) self-attention: each position may attend only to itself and earlier positions. That single constraint is the difference between BERT-style encoders and GPT-style causal models. Generation is then autoregressive: predict a token, append it, feed the whole thing back, repeat.

The second half is decoding, meaning how you pick a token from the final softmax over the vocabulary, and it matters more than people expect. Greedy decoding (always take the max) is deterministic, best for factual, math, and reasoning tasks and for anything where two users asking the same question should get the same answer. Pure sampling draws proportional to probability, which buys creative diversity but occasionally pulls from the long tail, and the key statistic is that although each individual tail token is unlikely, the tail in aggregate carries real mass (his worked example: 10%). Once an LLM samples a bad token it cannot recover, because the mistake becomes context for everything after. Temperature and top-k/top-p style controls (the sliders behind "creative vs precise" modes) manage that tradeoff.

The lecture ends with byte-pair encoding, the tokenization scheme that kills the UNK problem. Start with characters as tokens, repeatedly merge the most frequent adjacent pair into a new token, stop at a target vocabulary size (about 50k for GPT-2/3, 100k for GPT-4). The result mixes characters, frequent subwords like "ize", and frequent whole words, usually with a leading space baked in ("the" and " the" are different tokens), so any string ever typed, including invented words, tokenizes cleanly. His own first name was not worthy of a single token in uppercase, but "rama" lowercase was.

Lecture 10: Making GPT useful. Instruction tuning, RAG, LoRA

GPT-1, 2, and 3 differ only in scale, but scale produced a phase change: GPT-3 could write coherent long continuations, and it exhibited in-context learning (few-shot prompting from examples in the prompt), which nobody designed or predicted. What it could not do was follow instructions, because next-word prediction on internet text has no concept of an instruction; ask it to explain the moon landing and it continues with another question, pattern-matching a list. OpenAI's fix, from the InstructGPT paper he calls the breakthrough behind ChatGPT, has two stages. Supervised fine-tuning: about 12,500 human-written question-answer pairs, trained with plain next-word prediction. Then RLHF, motivated by economics: writing good answers is expensive but ranking answers is cheap, so humans rank model outputs, a reward model learns the rankings, and reinforcement learning pushes the LLM toward preferred outputs.

The middle section is a practitioner's tour of prompting (roles, "I will tip you", few-shot grammar correction) and retrieval augmented generation. RAG exists because of the context window: prompt plus output cannot exceed a fixed token budget, and chat history is re-fed on every turn, so you cannot paste the whole FAQ. Instead, chunk your documents, embed each chunk, embed the incoming question, retrieve the most similar chunks, and pack only those into the prompt. He calls RAG the most prevalent business application of LLMs he has seen.

Then the fine-tuning economics that make the LoRA section land. Llama-2 70B: roughly 140 GB just to hold weights at 2 bytes each, several multiples more for gradients and optimizer state, 6-7 A100s to fit it, and about $4M of optimistic GPU time to pretrain on 2 trillion tokens. Fine-tuning your use case needs only hundreds to thousands of examples, so data is not the problem; memory is. Gradient checkpointing trades compute for gradient memory. LoRA handles the optimizer state: freeze everything, write the fine-tune's change to each attention matrix as the product of two skinny low-rank matrices, and train only those. For an 8k x 8k matrix that is 0.02% of the original parameters, and a 7B or 13B model fine-tunes on one Colab GPU.

Lecture 11: Diffusion models and text-to-image

The generative image problem, split in two: generate any image from the training distribution, then steer generation with text. Part one uses the reverse-the-question trick he attributes to Polya's How to Solve It. Generating an image from noise is baffling, but the reverse is trivial: take Killian Court, add per-pixel Gaussian noise repeatedly until it is unrecognizable. That gives unlimited supervised pairs (noisier image in, less noisy image out), so train a network to denoise one step, then generate by starting from pure noise and applying the denoiser repeatedly (Stable Diffusion's pipeline defaults to 50 steps). Fresh random noise each time yields a different image, but always from the trained distribution: college buildings, not Labradors.

The denoiser is a U-Net, the standard image-to-image architecture: convolutions downsample to a compact representation, then upsample back to full size, with skip connections gluing each downsampling stage's output to its mirror-image upsampling stage so late layers see early detail. Same architecture for colorization, super-resolution, and segmentation.

Part two is conditioning, and the key ingredient is CLIP: a text encoder (transformer) and an image encoder (ResNet-like) trained jointly so that matching caption-image pairs get nearby embeddings and mismatched pairs get pushed apart. Once text and images share an embedding space, a prompt's embedding sits near the images it describes, and the denoiser can be conditioned on it. The Colab runs Stable Diffusion via the Hugging Face diffusers library (astronaut on a horse, an MIT professor on a horse by class request, negative prompting to delete the blue scarf from a Vermeer-style Labrador), then closes the course with off-the-shelf vision pipelines: classification, object detection with bounding boxes, and segmentation masks, all a few lines each.

Threads that run through the whole course

  • Start with the simplest model that could work and only add complexity when it stops working. This is stated in lecture 2 and obeyed in every Colab.
  • Hand-count parameters until the mechanics are reflexive. It pays off when transformer blocks arrive.
  • ReLU for hidden layers, output activation dictated by the output type, Adam as optimizer, dropout when overfitting. These defaults get you surprisingly far.
  • Everything trains the same way: define a loss, get gradients, run SGD. Heart disease MLP, GPT, AlphaFold, GloVe, diffusion denoisers. Only the architecture and data change.
  • Pretrained models beat training from scratch when data is small, and lose when data is large. Shown twice with numbers: ResNet on 98 handbag photos, GloVe versus scratch embeddings on 50k lyrics.
  • The transformer block never changes across domains. New modality means new tokenization into embeddings, nothing else.
