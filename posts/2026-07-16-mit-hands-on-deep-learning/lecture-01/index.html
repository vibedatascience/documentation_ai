The problem

AI was founded at the 1956 Dartmouth workshop (Minsky, McCarthy, and Shannon were all MIT), and the founders expected the field to be substantially solved within months. The slide framing is that the decades since have produced a sequence of breakthroughs, each one an answer to the previous approach's failure:

traditional AI interview experts, hand-write if-then rules │ fails: Polanyi's paradox, no generalization ▼ machine learning learn input→output mapping from examples │ fails on unstructured data (images, text, audio) ▼ deep learning learns the representation itself, automatically │ ▼ generative AI deep learning that produces unstructured output

Rule-based AI failed for two reasons. First, Polanyi's paradox: we know more than we can tell. You classify dog versus cat in about 20 milliseconds, but you cannot articulate how, and when you try, your explanation does not match what your brain actually does. If the expert cannot state the rule, you cannot program it. Second, hand-written rules are a small sample of an infinite world, so they break on situations nobody anticipated.

Machine learning (learn a mapping from input-output pairs using statistics; linear regression counts) fixed that, but only for structured data, meaning data that fits in spreadsheet columns. An image is three tables of numbers, one each for red, green, and blue light, every value 0 to 255. A 251 in the blue table means "lots of blue light here" whether the object is sky, water, or paint. The raw numbers carry no intrinsic meaning, so no statistical algorithm can use them directly. The historical workaround was manual feature engineering: measure beak length, wingspan, and primary color, and build a spreadsheet from the picture. That is a representation of the raw data, and building representations by hand was the human bottleneck. Whole PhDs went into a representation for one narrow problem, like detecting one stroke type in CT scans.

The idea

Deep learning learns the representations automatically from the raw input, then pipes them into what is basically a linear or logistic regression at the end. Three forces converged to make the old neural network idea finally work: new algorithmic tricks, large datasets, and GPUs.

The business consequence he pushes: every sensor in the world can now detect, recognize, and classify what it senses. Face ID is a camera plus an image classifier. Regina Barzilay's mammogram model at MGH flagged a problem in her own years-old scan that a radiologist had cleared. Smart binoculars that name the bird you are looking at shipped two weeks before the lecture. His startup heuristic: find a sensor, ask what a model behind it could do. Generative AI is the output-side unlock: models could always emit a few numbers (a probability, ten class scores, GPS coordinates), but generating pictures, audio, and text is new, and multimodal models that mix them in one conversation are becoming the norm.

The mechanics

Logistic regression, redrawn as a graph. The worked example is a hiring dataset: two inputs (GPA, years of experience), binary label (called for interview or not). Fitting logistic regression with any standard tool gives intercept 0.4 and coefficients 0.2 (GPA) and 0.5 (experience). Redraw the formula as a graph:

GPA ────0.2───┐ ├──▶ (+) ──▶ σ ──▶ P(interview) exp ────0.5───┘ ▲ │ 0.4 (bias)

Numbers flow left to right, get multiplied by their weight on each arrow, summed at the (+) node with the bias, then squashed by the sigmoid σ(a) = 1/(1+e-a). For an applicant with GPA 3.8 and 1.2 years of experience: 0.4 + 0.2(3.8) + 0.5(1.2) = 1.76, and σ(1.76) ≈ 0.85. Terminology swap for the neural net world: coefficients are weights, intercepts are biases, and "leaked model weights" in the news means exactly these numbers, since weights plus architecture reconstruct the model.

Why bother with the graph? Because once it is drawn, the obvious move appears: you can put things in the middle. The input is fixed by the problem and the output is fixed by the problem, so the only agency you have is between them. The most basic thing to do with any data is run it through a linear function, so insert a stack of linear functions: k inputs compress to, say, 3 numbers, then again, and again. Each transformation is capacity to learn something interesting, and this middle section is exactly where the automatic representation learning lives.

The nonlinearity. There is a catch: stacked linear functions collapse to one linear function, so without a nonlinearity between layers the entire effort is wasted. Each node therefore applies an activation function, a scalar-in scalar-out function, after its linear sum. The menu:

ActivationFormulaUse
ReLUmax(0, a)default for all hidden layers
Sigmoid1/(1+e-a)output layer, single probability
Softmaxea_i/Σea_joutput layer, multi-class (lecture 3)
Linearaoutput layer, plain number

The ReLU passes positive numbers through unchanged and squashes negatives to zero, and many practitioners credit it as a key reason deep learning took off (the gradient story behind that lands in lecture 2). There is a menagerie of alternatives (tanh, leaky ReLU, GELU, swish) you can ignore in practice.

Names for everything. A neuron is a linear function followed by an activation: many numbers in, one number out. A vertical stack of neurons is a layer. Input layer and output layer are given; everything between is a hidden layer. If every neuron's output feeds every neuron in the next layer, the layer is fully connected or dense. Logistic regression is a neural network with zero hidden layers, and deep learning is neural networks with many. Data flowing strictly left to right makes it a feedforward (or vanilla) network, and the whole arrangement of layers, sizes, activations, and connections is the architecture. Transformers and CNNs, coming later, are particular architectures.

The concept check. Build the interview model as a real network: 2 inputs, one hidden layer of 3 ReLUs, sigmoid output.

x1 ──┬──▶ ReLU ──┐ ├──▶ ReLU ──┼──▶ σ ──▶ y x2 ──┴──▶ ReLU ──┘ 2·3 weights 3·1 weights + 3 biases + 1 bias = 13 parameters

The class guessed 12 and 9 before landing on 13, because everyone forgets the biases. His standing instruction: hand-count parameters for your first few networks until it is reflexive, because transformers will be hard if the mechanics are not automatic. Slides 100 through 104 then assume the network has been trained (details deferred to lecture 2) and walk one input through it by hand. These are the actual weights from the slides:

a1 = max(0, -0.3 + 0.5*x1 + 0.1*x2)
a2 = max(0,  0.2 - 0.1*x1 + 0.3*x2)
a3 = max(0,  0.5 + 0.2*x1 - 0.1*x2)
y  = sigmoid(0.05 - 0.2*a1 - 0.3*a2 - 0.15*a3)

At the slide defaults x = (2.3, 10.2), the bottom neuron's linear sum is negative, the ReLU zeroes it out, and y lands at 0.226, exactly the number on slide 104. Drag the sliders and watch neurons die and revive as their sums cross zero. Written out as a single formula (slide 105 does it), even this 13-parameter network is an unreadable nest of maxes and exponentials, far more complicated than the logistic regression it grew from, with the same two inputs. That complexity is where the capability comes from, provided you can train it.

Design choices

The closing summary slide draws the line explicitly. Dictated by the problem: the input layer (one node per independent variable) and the output layer (its size and activation follow from what you are predicting). Yours to choose: the number of hidden layers, the number of neurons in each, and the hidden activation functions. The course default for that last one is ReLU, no deep thought needed. How to choose the rest is a running theme of the next lectures, and the general answer previewed here is experimentation, not theory.

Sharp edges

  • Parameter counts hide the biases. Every wrong guess in class (12, 9) missed them. Count weights as products of adjacent layer sizes, then add one bias per non-input neuron.
  • Interpretability, from Q&A: you always know exactly how a data point flows through the network, every multiplication is visible, but you cannot easily credit any single input for the outcome, because everything gets enmeshed in the middle layers.
  • A hidden layer without an activation adds nothing. Two stacked linear functions are one linear function, so the network silently degrades to logistic regression with extra steps.
  • The output activation is not a style choice. Probability means sigmoid, plain number means linear. Mismatching it breaks the model's meaning before training even starts.

Sets up

The forward pass assumed someone handed us trained weights. Lecture 2 answers where they come from: define a loss function measuring how bad the current weights are, then walk downhill on it with gradient descent, minibatch by minibatch, gradients computed via backprop.
