Turn long YouTube videos into narrated slideshows

I don't have 90 minutes for every podcast or lecture I want to get through, so I built a pipeline that condenses them into short videos I can watch on the subway. The best run so far took a 31-minute Ken Burns interview down to 3.5 minutes with the paintings he talks about as slide backgrounds. Nothing in the stack is exotic: a transcript API, an LLM, PIL, edge-tts and ffmpeg.

YouTube URL │ ▼ [1] transcript EasySubAPI (or yt-dlp + Whisper) │ ▼ [2] distill LLM condenses to 8-20 slide script │ ▼ [3] slides 1920×1080 cards + real imagery/diagrams │ ▼ [4] voiceover edge-tts, one clip per slide │ ▼ [5] render ffmpeg: zoompan + fades → final.mp4

Get the transcript

For YouTube, a transcript API is faster than downloading the video:

import http.client, json

conn = http.client.HTTPSConnection("easysubapi.p.rapidapi.com")
payload = json.dumps({"video_id": "VIDEO_ID"})
headers = {
    "x-rapidapi-key": "YOUR_KEY",
    "x-rapidapi-host": "easysubapi.p.rapidapi.com",
    "Content-Type": "application/json"
}
conn.request("POST", "/api/easysubapi-get-transcript", payload, headers)
data = json.loads(conn.getresponse().read())
frames = data["result"][0]["data"]["frames"]

For non-YouTube sources, extract audio with ffmpeg and transcribe locally with faster-whisper.

Clean and distill

Auto-captions repeat every phrase two to four times across frames and arrive full of HTML tags. Exact-match dedup is not enough. Strip tags, unescape, sentence-split, drop consecutive duplicates, dedupe:

import re, html

text = " ".join(re.sub(r"<[^>]+>", "", html.unescape(f["message"])) for f in frames)
sents = re.split(r"(?<=[.!?]) +", text)
sents = [s for i, s in enumerate(sents) if i == 0 or s != sents[i-1]]
clean = " ".join(dict.fromkeys(sents))

Then the LLM condenses the clean transcript into a slide script: 8-20 slides, each with a headline, 2-3 lines of narration and an image direction. This step decides the quality of the whole thing. Prompt for core ideas and the best verbatim quotes, not a flat summary.

Build the slides

1920×1080 PNGs via PIL. The first version was dark text cards. Technically fine, deadly boring. Three changes fixed it:

  • Full-bleed imagery matched to each slide: public domain paintings for a history podcast, stadium photography for football, custom diagrams for a lecture
  • Cinematic grading: darken and desaturate so white text sits on top
  • Programmatic diagrams (PIL/matplotlib) for technical content: network architectures, tactical boards, activation curves. Beats hunting for stock

Voiceover

edge-tts, Microsoft's neural voices. Pip-installable, free, good. One MP3 per slide, so slide timing is driven by audio duration. No manual sync.

pip install edge-tts
edge-tts --voice en-US-GuyNeural --text "$NARRATION" --write-media slide_01.mp3

en-US-GuyNeural has a documentary feel. en-GB-RyanNeural when the source is British.

Render

Each slide pairs its PNG with its MP3, gets a slow zoompan (alternate in and out between slides), crossfades, then concatenates:

ffmpeg -loop 1 -i slide_01.png -i slide_01.mp3 \
  -vf "zoompan=z='min(zoom+0.0008,1.08)':d=125*10:s=1920x1080,fps=25" \
  -c:v libx264 -pix_fmt yuv420p -shortest seg_01.mp4

ffmpeg -f concat -safe 0 -i segments.txt -c copy final.mp4

Gotchas

GOTCHA · ROLLING CAPTIONS

Strip tags, unescape, sentence-split, then dedupe. In that order.

GOTCHA · TLS + EDGE-TTS

Behind an egress proxy with a self-signed cert, edge-tts's websocket TLS verification fails. Bypass verification for that one connection only.

GOTCHA · PLAIN CARDS ARE BORING

Text-on-dark-card slides feel like a settings screen. Imagery per slide is the difference between "delete" and "rewatch".

GOTCHA · ONE SESSION

Ephemeral sandboxes reset. Run the whole pipeline in one go and save intermediates as you go.
