Turn long YouTube videos into narrated slideshows

I don't have 90 minutes for every podcast or lecture I want to get through, so I built a pipeline that condenses them into short videos I can watch on the subway. Nothing in the stack is exotic. Each step has options depending on the content and what you have access to.

YouTube URL / video file │ ▼ [1] transcript A. transcript API (YouTube) │ B. ffmpeg audio + faster-whisper (any video) ▼ [2] script any capable LLM → strict JSON slide script │ adaptive slide count, 55-80 words/slide ▼ [3] visuals A. AI-generated painterly imagery (narrative) │ B. white lecture slides + matplotlib (technical) │ composited to 1920×1080 PNGs via PIL ▼ [4] voiceover A. edge-tts (free, zero setup) │ B. steerable TTS e.g. gpt-4o-mini-tts ▼ [5] render raw ffmpeg: zoompan + fades + score → final.mp4

1. Transcript

Option A, YouTube: a transcript API is faster than downloading the video.

import requests

r = requests.post(
    "https://easysubapi.p.rapidapi.com/api/easysubapi-get-transcript",
    json={"video_id": "VIDEO_ID"},
    headers={"x-rapidapi-key": "YOUR_KEY"})
frames = r.json()["result"][0]["data"]["frames"]

A 56-minute lecture is roughly 1,365 frames / 56k characters. Fits in a single LLM call with the script prompt.

Option B, any video file: extract audio with ffmpeg, transcribe locally with faster-whisper (tiny or base model, CPU int8 is fine).

Auto-captions repeat every phrase two to four times and arrive full of HTML tags. Clean before use:

import re, html

text = " ".join(re.sub(r"<[^>]+>", "", html.unescape(f["message"])) for f in frames)
sents = re.split(r"(?<=[.!?]) +", text)
sents = [s for i, s in enumerate(sents) if i == 0 or s != sents[i-1]]
clean = " ".join(dict.fromkeys(sents))

2. Script

Any capable LLM works. Ask for strict JSON: title, subtitle, slides, each with heading, key points, narration, a tone direction for the TTS, and a diagram or image direction. Strip the ``` fences before json.loads.

Three prompt decisions matter more than the model:

  • Adaptive length. Don't fix the slide count. Let the model propose it from the content's density: a light 20-min talk might need 6-8 slides, a dense 90-min lecture 15-20. Cap narration at 55-80 words per slide.
  • Keep the speaker's voice. Prompt the model to lift the speaker's own phrasings, quirks and asides verbatim from the transcript, lightly trimmed. The result should feel like the original speaker condensed, not a new narrator paraphrasing.
  • Core ideas and best quotes, not a flat summary.

3. Visuals

Option A, painterly imagery, for narrative content. An image model per slide. Style suffix that worked: "oil painting style, dramatic chiaroscuro lighting, muted palette, cinematic, no text, no words, no letters". Parallelize the calls. Composite in PIL: cover-crop the image to 1920×1080, dark gradient at the bottom, chapter label, wrapped headline, pull-quote.

Option B, lecture slides, for technical content. White template: heading, bullets on the left, a real matplotlib diagram on the right in a bordered card. Real plots, not generated images: activation curves, loss contours, network graphs, timelines. Give the script model a fixed diagram vocabulary and let it pick one per slide, each used at most once.

Either way, the first version I made was plain dark text cards, which were technically fine but deadly boring. Imagery per slide is what makes it watchable.

4. Voiceover

Option A, edge-tts. These are Microsoft's neural voices, and they are free, pip-installable, and need zero setup. en-US-GuyNeural has a documentary feel; en-GB-RyanNeural when the source is British.

pip install edge-tts
edge-tts --voice en-US-GuyNeural --text "$NARRATION" --write-media slide_01.mp3

Option B, steerable TTS. If you have an OpenAI key or an internal LLM proxy, gpt-4o-mini-tts takes an instructions field ("documentary narrator", "thinking out loud, uneven pacing") plus the per-slide tone from the script model. The instructions change delivery more than the voice choice does. Naturalness comes from writing fillers into the input text itself: ellipses, fragments, "so...", "right?". Keep them sparse at 1-2 per slide.

One clip per slide either way, so slide timing is driven by audio duration and nothing needs manual sync.

5. Render

Raw ffmpeg, one segment per slide: pair the PNG with its MP3, apply a slow zoompan, fade, then concat.

ffmpeg -loop 1 -i slide_01.png -i slide_01.mp3 \
  -filter_complex "[0:v]scale=1920:1080,zoompan=z='1+0.12*on/FRAMES':x='(iw-iw/zoom)/2':y='(ih-ih/zoom)/2':d=FRAMES:s=1280x720:fps=24[v]" \
  -map "[v]" -map 1:a -t DUR seg_01.mp4

ffmpeg -f concat -safe 0 -i segments.txt -c copy final.mp4

Tuning that took iteration:

  • Alternate zoom in and zoom out between slides. Segment duration = narration + 1s.
  • 12% zoom suits full-bleed imagery. Text slides need 4% or the words swim.
  • Fade each segment in/out 0.5s (video and audio), then concat with the demuxer.
  • Ambient score under the narration: synthesize in numpy (low sine stack + LFO + noise, faded), mix with amix. Volume 0.30 under imagery, 0.14 under lecture slides.

Gotchas

GOTCHA · ROLLING CAPTIONS

Strip tags, unescape, sentence-split, then dedupe, in that order.

GOTCHA · MOVIEPY IS SLOW

moviepy timed out at 600s rendering what raw ffmpeg zoompan did in 13s. Use ffmpeg subprocess calls directly.

GOTCHA · TLS + EDGE-TTS

Behind an egress proxy with a self-signed cert, edge-tts's websocket TLS verification fails. Bypass verification for that one connection only.

GOTCHA · SANDBOX SUBPROCESS GUARDS

Some sandboxes block subprocess commands built from variables or f-strings. Compute durations and frame counts first, print them, then emit fully literal ffmpeg commands. It is tedious but reliable.

GOTCHA · PATHS MOVE

Output dirs can change if the session restarts. Never hardcode tmp paths across cells; re-check and re-glob each cell. Run the whole pipeline in one go and save intermediates as you go.
