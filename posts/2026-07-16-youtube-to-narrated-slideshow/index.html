[2026-07-16]LLM-APPS

Turn long YouTube videos into narrated slideshows

What I built

A pipeline that takes a long YouTube video — a 31-minute podcast interview, a 90-minute football pod, an MIT lecture — and produces a short, watchable MP4: designed slides, full-bleed imagery, neural voiceover, and slow Ken Burns zooms. A 31-minute interview becomes a 3.5-minute video. An hour-long lecture becomes an 11-minute, 19-chapter explainer with custom diagrams.

YouTube URL │ ▼ [1] transcript EasySubAPI (or yt-dlp + Whisper) │ ▼ [2] distill LLM condenses to 8–20 slide script │ ▼ [3] slides 1920×1080 cards + real imagery/diagrams │ ▼ [4] voiceover edge-tts, one clip per slide │ ▼ [5] render ffmpeg: zoompan + fades → final.mp4

Why

Long-form content is where the good stuff is, but I don't have 90 minutes. Passive summaries (bullet points) lose the narrative. A condensed video keeps the ideas in order, keeps the quotes, and is something I'll actually watch on the subway.

Steps

1 · Get the transcript

For YouTube, a transcript API is faster than downloading the video. I use EasySubAPI via RapidAPI:

import http.client, json

conn = http.client.HTTPSConnection("easysubapi.p.rapidapi.com")
payload = json.dumps({"video_id": "VIDEO_ID"})
headers = {
    "x-rapidapi-key": "YOUR_KEY",
    "x-rapidapi-host": "easysubapi.p.rapidapi.com",
    "Content-Type": "application/json"
}
conn.request("POST", "/api/easysubapi-get-transcript", payload, headers)
data = json.loads(conn.getresponse().read())
frames = data["result"][0]["data"]["frames"]

Fallback for non-YouTube sources: extract audio with ffmpeg and transcribe locally with faster-whisper.

2 · Clean and distill

Rolling captions repeat every phrase across consecutive frames, so exact-match dedup isn't enough. Split into sentences, drop consecutive duplicates, then dedupe order-preservingly:

import re, html

text = " ".join(re.sub(r"<[^>]+>", "", html.unescape(f["message"])) for f in frames)
sents = re.split(r"(?<=[.!?]) +", text)
sents = [s for i, s in enumerate(sents) if i == 0 or s != sents[i-1]]
clean = " ".join(dict.fromkeys(sents))

Then the LLM step: condense the clean transcript into a slide script — 8–20 slides, each with a headline, 2–3 lines of narration, and an image/diagram direction. This is where the quality lives. Prompt for the core ideas and best verbatim quotes, not a flat summary.

3 · Build the slides

1920×1080 PNGs via PIL. The first version I made was plain dark text cards — technically fine, deadly boring. What fixed it:

  • Full-bleed imagery matched to each slide's content — public domain paintings for a history podcast, stadium photography for football, custom-drawn diagrams for a lecture
  • Cinematic grading — darken + slight desaturation so white text sits on top
  • One accent color, pull quotes set large, image credits small in the corner
  • For technical content, draw the diagrams programmatically (PIL/matplotlib) — network architectures, tactical boards, activation curves — instead of hunting for stock

4 · Voiceover

edge-tts (Microsoft neural voices, pip-installable, free) generates one MP3 per slide. Quality is genuinely good.

pip install edge-tts
edge-tts --voice en-US-GuyNeural --text "$NARRATION" --write-media slide_01.mp3

Voice picks: en-US-GuyNeural for a documentary feel, British voices (en-GB-RyanNeural) when the source is British. One clip per slide means slide timing is driven by audio duration — no manual sync.

5 · Render

ffmpeg per slide: pair the PNG with its MP3, apply a slow zoompan (alternate zoom-in and zoom-out between slides), crossfade, then concat:

ffmpeg -loop 1 -i slide_01.png -i slide_01.mp3 \
  -vf "zoompan=z='min(zoom+0.0008,1.08)':d=125*10:s=1920x1080,fps=25" \
  -c:v libx264 -pix_fmt yuv420p -shortest seg_01.mp4

ffmpeg -f concat -safe 0 -i segments.txt -c copy final.mp4

Gotchas

GOTCHA · ROLLING CAPTIONS

YouTube auto-captions repeat each phrase 2–4× across frames, and messages contain HTML tags and entities. Strip tags, unescape, sentence-split, then dedupe — in that order.

GOTCHA · TLS + EDGE-TTS

Behind a corporate/egress proxy with a self-signed cert, edge-tts's websocket TLS verification fails. Bypass verification for that one connection only.

GOTCHA · PLAIN CARDS ARE BORING

Text-on-dark-card slides feel like a settings screen. Imagery per slide is not optional — it's the difference between "delete" and "rewatch".

GOTCHA · ONE SESSION

If running in an ephemeral sandbox, the environment resets — run the whole pipeline in one go, and save intermediates (transcript, script JSON, slide PNGs) as you go.
