A zoomable treemap canvas for world population

I saw a treemap of online attention where the top sites are labeled and the long tail is a field of confetti: 344,753 websites from a browser tracking panel, June 2024 to December 2025. I wanted the same idea as an infinite canvas: one flat map you can scroll into anywhere, with labels that appear as tiles get big enough to hold them. This one shows the 2024 population of every country, all 217 of them, 8.12B people.

open full screen →

Scroll to zoom, drag to pan, double-click a tile to fly to it. The toggle switches between flat colors, coloring by World Bank region, and a full regrouped layout with region blocks. India and China each hold about 18% of the map. Tuvalu is 9,646 people in the corner.

Data

Everything comes from the World Bank API, no key required. One call for the populations and one for country metadata to drop the aggregate rows like "Africa Eastern and Southern" that the indicator endpoint mixes in with real countries.

import urllib.request, json

meta = json.load(urllib.request.urlopen(
    "https://api.worldbank.org/v2/country?format=json&per_page=400"))[1]
real = {c["id"]: c["region"]["value"] for c in meta
        if c["region"]["value"] != "Aggregates"}

pop = json.load(urllib.request.urlopen(
    "https://api.worldbank.org/v2/country/all/indicator/SP.POP.TOTL"
    "?format=json&per_page=400&date=2024"))[1]
rows = [{"name": r["country"]["value"], "pop": r["value"],
         "region": real[r["countryiso3code"]]}
        for r in pop if r["countryiso3code"] in real and r["value"]]

The rows get embedded straight into the HTML as a JSON literal, so the page works offline and there is no fetch at load time.

The canvas

The layout is a single d3.treemap in a fixed 1400x950 coordinate space. d3.zoom applies a transform to one group element, scale extent 0.8 to 300. That range matters here because the sizes span five orders of magnitude, from 1.45B down to under 10K.

The trick that makes it feel like a canvas is counter-scaled text. Tile geometry scales with the zoom transform but label font sizes are divided by the zoom factor k, so text stays a constant size on screen while more of it becomes visible.

function updateLabels() {
  leaves.each(function(d) {
    const w = (d.x1 - d.x0) * k;   // on-screen width
    const h = (d.y1 - d.y0) * k;
    let px = Math.min(26, Math.max(11, Math.sqrt(w * h) / 7));
    const show = px >= 9 && h > px * 1.7 && w > 34;
    d3.select(this).select("text")
      .style("display", show ? null : "none")
      .style("font-size", (px / k) + "px");  // counter-scale
  });
}

Each tile carries three text lines: the country name, the population, then share and rank. Each line has its own visibility threshold based on on-screen height, so tiles gain detail progressively as you zoom instead of all at once. Stroke widths are divided by k too, otherwise borders turn into walls at 100x.

Mode switching

Both layouts, flat and grouped by region, are computed once up front and indexed by ISO code. Switching modes is just a 700ms transition of each tile to its position in the other layout. Nothing gets recomputed on toggle.

GOTCHA · SIZING
The first version computed the treemap size from container.clientWidth at script time and rendered completely blank inside an iframe, because the container measured zero before layout. Fixed coordinates plus an SVG viewBox avoid runtime measurement entirely and the browser handles scaling.

The pattern generalizes to anything with a name and one number where the ratios are extreme: every earthquake this year by energy released, every word in English by frequency, every crypto token by market cap. The zoom is only worth building when the biggest tile is 100,000x the smallest.
